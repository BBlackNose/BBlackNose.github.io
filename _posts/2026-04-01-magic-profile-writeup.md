# [Write-up] Magic Profile: SSTI 취약점 분석 및 익스플로잇

## 1. 문제 개요
- **문제 이름:** Magic Profile
- **분류:** Web Hacking (SSTI)
- **난이도:** Middle
- **목표:** 서버에 숨겨진 `flag.txt` 파일을 찾아 내용을 읽어낸다.

## 2. 취약점 탐색 (Reconnaissance)
해당 서비스는 사용자로부터 `name`과 `bio`를 입력받아 카드를 생성해주는 기능을 제공합니다. 입력 필드에 `{{7*7}}`과 같은 템플릿 문법을 삽입했을 때, 결과 화면에 **`49`**가 출력되는 것을 확인하여 **SSTI(Server-Side Template Injection)** 취약점이 존재함을 파악했습니다.

## 3. 보안 필터링 분석 (WAF/Filter Bypass)
서버는 보안을 위해 다음과 같은 문자열 및 패턴을 검사하고 차단(`Card generation blocked.`)하고 있었습니다.
*   **`.` (점):** 객체의 속성에 접근하는 표준 문법 차단.
*   **`__` (언더바):** 파이썬 시스템 속성(`__class__`, `__init__` 등) 접근 차단.
*   **키워드:** `config`, `self`, `request`, `lipsum`, `class`, `base` 등 주요 객체명 차단.

### 우회 전략 (Bypass Strategy)
1.  **점(`.`) 우회:** Jinja2의 **`|attr()`** 필터를 사용하여 `obj.attr` 대신 `obj|attr('attr')` 형식으로 접근.
2.  **언더바(`__`) 및 키워드 우회:** 문자열 전체를 **8진수 인코딩(Octal Encoding)** 처리하여 서버의 문자열 검사를 완전히 무력화. (예: `__class__` → `\137\137\143\154\141\163\163\137\137`)

## 4. 익스플로잇 과정 (Exploitation)

### 1단계: 하위 클래스 탐색 (Subclasses Enumeration)
먼저 서버에 로드된 모든 클래스 목록을 추출하여 시스템 명령어를 내릴 수 있는 도구를 찾습니다.
```jinja2
{{ []|attr('\137\137\143\154\141\163\163\137\137')|attr('\137\137\142\141\163\145\137\137')|attr('\137\137\163\165\142\143\154\141\163\163\145\163\137\137')() }}
```
추출된 목록 중 **138번** 인덱스에서 시스템 명령 실행에 유용한 **`os._wrap_close`** 클래스를 발견했습니다.

### 2단계: 파일 목록 확인 (RCE - ls)
발견한 138번 클래스의 `popen` 함수를 호출하여 현재 디렉토리의 파일 목록을 확인합니다. (명령어: `ls -R`)
```jinja2

```
결과 화면에서 **`flag.txt`** 파일의 존재를 확인했습니다.

### 3단계: 최종 플래그 탈취 (Flag Retrieval)
`cat flag.txt` 명령어를 실행하여 플래그를 화면에 출력합니다.

## 6. 결론 및 대응 방안
본 문제는 강력한 블랙리스트 필터링 환경에서도 파이썬 객체의 계층 구조를 이용해 시스템 권한을 획득할 수 있음을 보여줍니다. 템플릿 엔진 사용 시 다음과 같은 보안 대책이 필요합니다.
1. **사용자 입력값 검증:** 템플릿 구문(`{{`, `}}`) 및 특수문자에 대한 엄격한 필터링.
2. **Safe Template 사용:** 렌더링 시 위험한 함수나 객체에 접근할 수 없도록 제한된 환경 제공.
3. **최소 권한 원칙:** 웹 애플리케이션 실행 계층의 권한을 최소화하여 RCE 발생 시 피해 범위 축소.
